## Decision Question

What primary development workstation should be selected for the Crucible
Registry ecosystem?

The decision should identify a workstation that supports the user's actual
development, documentation, research, repository, and information-processing
workflow with strong ready-to-use productivity and long-term ownership value.

The workstation should also function as a stable work hub connecting the user's
work across different locations. It should house the repository working
environment and support the same level of work the user is doing now without
requiring supercomputer-class performance.

## Authority Boundary

The Decision Intelligence System supports this procurement decision.

It does not own the Crucible Registry project, its repositories, its operating
policies, or the final purchase authority.

The workstation is the execution environment.

The project repository remains the durable asset.

## Mission

Create a repeatable, evidence-based workstation procurement decision using
explicit standards rather than ad hoc recommendations.

The decision output should:

- Search the current market.
- Validate pricing.
- Detect active promotions.
- Score candidate systems.
- Compare alternatives.
- Produce a procurement report.
- Update or re-evaluate the recommendation as market conditions change.

## Procurement Philosophy

Optimize for:

- Ready-to-use productivity
- Workflow fit
- Total ownership value
- Wireless performance
- Reliability
- Stable repository hub behavior
- Backup, sync, and recovery fit
- Network stability
- Reasonable future flexibility

Do not optimize solely for lowest purchase price.
Do not optimize for maximum performance when practical hub reliability and cost
discipline are more important.

Completed system cost includes all required upgrades, accessories, shipping,

The $1,500 figure remains the normal hard target for recommendations.

Candidates between $1,500 and $2,000 may be shown only when they are materially
better and the incremental value is explicit.

Candidates above $2,000 are out of scope unless the user changes the budget.

## Preferred Specification

Preferred baseline:

- Intel Core Ultra 5, Intel Core i5 or better, AMD Ryzen 5 or better
- Prefer Intel Core Ultra 7, AMD Ryzen 7, or AMD Ryzen 9 when value justifies
- Windows 11 Pro
- 64 GB RAM preferred
- 2 TB NVMe storage preferred
- 1 TB NVMe storage acceptable at purchase when the system has a clear,
  low-risk path to 2 TB or more
- Wi-Fi 7 preferred
- Wi-Fi 6E acceptable
- Excellent real-world wireless performance
- Easy maintenance

Storage classification:

- 512 GB: too tight unless explicitly temporary
- 1 TB: acceptable starting point if expandable
- 2 TB: preferred comfort target
- 4 TB or more: useful only when local datasets, virtual machines, media, or
  other storage-heavy work justify it

Reliability target:

- Minimum acceptable expected useful life: 3 years
- Target expected useful life: 4-5 years
- Stretch outcome: 5+ years when achieved without overbuying

These preferences are not final pass-fail requirements until the scoring model
and mandatory requirements are defined.

## Evidence Categories

Evidence should be collected under these categories:

- Requirements: actual work, required software, constraints, and success
  conditions.
- Candidate options: systems, vendors, configurations, warranties, and upgrade
  paths.
- Technical fit: CPU, memory, storage, operating system, ports, thermals,
  maintainability, and workload suitability.
- Wireless: Wi-Fi version, chipset, antenna quality, reliability,
  upgradeability, and real-world performance evidence.
- Cost: current price, typical price, historical low, promotions, coupons,
  required upgrades, completed cost, and total ownership value.
- Availability: stock status, shipping time, lead time, delivery risk, and
  market timing.
- Lifecycle: support horizon, repairability, upgrade path, warranty, resale
  value, and expected useful life.
- Risk: compatibility gaps, vendor lock-in, reliability concerns, upgrade
  difficulty, return friction, and decision reversibility.
- Work hub fit: storage reliability, backup/sync compatibility, network
  stability, low-friction remote or multi-location access, uptime behavior,
  sleep/wake reliability, and recovery path if the device fails.
- External evidence: benchmarks, reviews, vendor documentation, warranty
  terms, market pricing, and comparable alternatives.

## Scoring Priorities

Primary priorities:

- Ready-to-use value
- Repository workstation suitability
- Repository hub suitability
- Workflow fit
- Reliability
- Wireless performance
- Backup, sync, and recovery fit
- 4-5 year useful life expectation

Secondary priorities:

- Performance
- Expandability
- Warranty
- Performance per dollar

The case needs a scoring model before candidates are ranked.

The scoring model should separate mandatory requirements from weighted
preferences and should record unknowns explicitly.

## Upgrade Rules

Upgradeable systems remain eligible.

For each upgradeable candidate, record:

- Required upgrades
- Upgrade cost
- Upgrade difficulty
- Upgrade time
- Whether the upgrades are justified
- Estimated completed cost
- Time to productivity

Upgradeable candidates should not be treated as ready-to-use unless all
required upgrades are already included or can be completed without meaningful
delay or risk.

## Price Snapshot Requirements

For every candidate, collect:

- Current price
- Typical price
- Historical low
- Current promotion
- Coupon availability
- Ready-to-use status
- Required upgrades
- Estimated completed cost
- Time to productivity

Because pricing and promotions change quickly, price evidence must include
collection date and source.

## Monitoring Triggers

The recommendation should be re-evaluated when:

- A selected candidate changes price materially.
- A preferred configuration goes out of stock.
- A new promotion appears.
- A better candidate becomes available.
- Requirements change.
- Evidence about reliability, wireless performance, support, or upgradeability
  changes.
- The recommendation reaches its review date.

## Open Work

After order:

- Record final order total, taxes, shipping, delivery estimate, and return
  deadline when available.
- Preserve receipt and warranty evidence.
- Track shipment.
- Prepare first-boot and setup checklist.
- Validate delivered hardware against order.
- Confirm Windows 11 Pro activation.
- Confirm Wi-Fi, Ethernet, USB4, storage, thermals, noise, and sleep/wake
  behavior.
- Configure GitHub/repository hub workflow.
- Make post-arrival acceptance or return decision before the return window
  closes.

# Development Workstation Advisor v1.0

## Codex Project Charter (Bootstrap)

> This document is the initial governing specification for the
> Development Workstation Advisor project. It defines the project's
> purpose, scope, execution philosophy, and the standards Codex should
> use while building the remainder of the repository.

# Mission

Create a repeatable, evidence-based procurement system that evaluates
development workstations using explicit standards rather than ad hoc
recommendations.

The system shall produce consistent procurement reports, monitor the
market, compare alternatives, and recommend the best workstation for the
user's actual workflow.

The project is repository-first.

The repository is the durable asset.

The workstation is the execution environment.

# Primary Objective

Design a reusable procurement engine that:

-   Searches the current market.
-   Validates pricing.
-   Detects active promotions.
-   Scores candidate systems.
-   Produces executive procurement reports.
-   Updates recommendations as market conditions change.

# Scope

The first procurement domain is:

-   Development workstations

Future domains may include:

-   Monitors
-   NAS
-   Networking
-   Docking stations
-   Backup
-   Laptops
-   Peripherals

The architecture must support expansion without redesign.

# User Workflow

Primary software:

-   ChatGPT
-   Codex
-   VS Code
-   GitHub
-   Markdown repositories
-   Microsoft Office

Primary work:

-   Repository management
-   Documentation
-   Research
-   Software development
-   Information processing

# Procurement Philosophy

Optimize for: 1. Ready-to-use productivity. 2. Workflow fit. 3. Total
ownership value. 4. Wireless performance. 5. Reliability. 6. Reasonable
future flexibility.

Do NOT optimize solely for lowest purchase price.

# Budget

Preferred completed system: - \~\$1,000

Hard ceiling: - \$1,500

Completed system cost includes all required upgrades.

# Preferred Specification

-   Intel Core Ultra 5/Core i5+ or AMD Ryzen 5+
-   Prefer Core Ultra 7 / Ryzen 7 / Ryzen 9 when value justifies.
-   Windows 11 Pro
-   64 GB RAM preferred
-   2 TB NVMe preferred
-   Wi-Fi 7 preferred
-   Wi-Fi 6E acceptable
-   Excellent real-world wireless
-   Easy maintenance

# Evaluation Categories

Overall Top 10

Then provide:

-   Top 3 Ready-to-Use
-   Top 3 Upgrade Candidates
-   Top 3 Mini PCs
-   Top 3 Desktops / Workstations

# Report Requirements

Every report shall contain:

1.  Executive Summary
2.  Purchase Readiness Score (0-100)
3.  Current Market Summary
4.  Current Price Snapshot
5.  Top 10 Overall Recommendations
6.  Category Rankings
7.  Category Winners
8.  Active Promotions
9.  Historical Pricing
10. Incremental Value Analysis
11. Procurement Decision

# Current Price Snapshot

For every candidate include:

-   Current price
-   Typical price
-   Historical low
-   Current promotion
-   Coupon availability
-   Ready-to-use status
-   Required upgrades
-   Estimated completed cost
-   Time to productivity

# Weighting Priorities

Primary: - Ready-to-use value - Repository workstation suitability -
Workflow fit - Reliability - Wireless

Secondary: - Performance - Expandability - Warranty - Performance per
dollar

# Upgrade Rules

Upgradeable systems remain eligible.

They must state:

-   Required upgrades
-   Cost
-   Difficulty
-   Time
-   Whether upgrades are justified.

# Wireless Requirements

Wireless is a primary requirement.

Evaluate:

-   Wi-Fi version
-   Chipset
-   Antenna quality
-   Reliability
-   Upgradeability

# Procurement Decision

End every report with:

-   Buy Now
-   Wait
-   Why

Include confidence and rationale.

# Long-term Vision

This repository becomes the permanent procurement standard consumed by
Codex.

Future changes should modify standards rather than prompts.

The repository defines behavior.

Codex executes behavior.
